# スカウトメール開封率チェッカー - グルメキャリー

グルメキャリー管理画面（kanri.gourmetcaree.jp）のスカウトメール送信一覧で、
**ボタンひとつ**で全ページを自動で読み取り、開封状況を多角的に分析する
Chrome 拡張機能です。

「スカウトメール自動生成」拡張とは**完全に独立した別の拡張機能**です。
このフォルダ（`open-rate-checker`）だけで動作し、単体で別リポジトリに
移動してもそのまま使えます。

---

## できること

- メールBOXの一覧ページ（スカウトメールのほか、応募メールなど各タブでも可）の
  右下に「**📊 開封率チェッカー**」ボタンを表示
- クリックすると、いま表示しているタブの全ページ（1〜最終ページ）を自動で巡回して集計
- **ダッシュボード表示**:
  - 送信数／開封数／**開封率**／開封までの時間（中央値）のサマリー
  - **期間フィルタ**: 送信日で絞り込み（カレンダー指定＋「過去7日」「過去30日」「今月」プリセット）
  - **開封が多い時間帯**（0〜23時のグラフ）と**曜日**
  - **送信時間帯別・送信曜日別の開封率**（＝いつ送ると読まれやすいか）
  - **送信から開封までの経過時間**: 平均・中央値・分布（30分以内〜3日以上）
  - **件名別の開封率**（件名A/Bテストの効果測定に）
  - **送信日別の開封率**（日ごとの推移）
  - **💡 自動インサイト**: 「開封が最も多いのは12時台」「火曜送信の開封率が最も高い」
    「開封の大半は24時間以内 → 翌日未読なら開封見込み薄」「3日以上未読が◯件（再アプローチ候補）」
    など、数字から読み取れる気づきを文章で提示
- **エクスポート**: 明細（状態・送信日時・開封日時・開封までの分数・件名・宛先）を
  タブ区切りでコピー（Excel/スプレッドシートに直接貼り付け可）、またはCSVダウンロード（Excel対応のBOM付きUTF-8）
- 読み取り専用です。**サイト上のデータを変更したり、メールを送信したりすることは一切ありません**

## インストール方法（初回のみ・約3分）

1. リポジトリの緑色の「**Code**」ボタン →「**Download ZIP**」でダウンロード
2. ZIPファイルを右クリック →「すべて展開」（好きな場所でOK。**展開後のフォルダは消さないこと**）
3. Chrome のアドレスバーに `chrome://extensions` と入力して開く
4. 右上の「**デベロッパーモード**」をONにする
5. 「**パッケージ化されていない拡張機能を読み込む**」をクリック
6. 展開した中の **open-rate-checker フォルダ** を選択
7. 一覧に「スカウトメール開封率チェッカー - グルメキャリー」が表示されたら完了

> 更新版が出たときは、新しいZIPを同じ場所に展開し直して、
> `chrome://extensions` で拡張機能の「🔄（更新）」を押してください。

## 使い方

1. 管理画面の **メールBOX** を開き、**「送信」** を選んでから **「スカウトメール」タブ** をクリック
   （受信のまま実行すると受信メールを数えてしまいます。
   応募メールなど他のタブで実行すれば、そのタブの開封率を集計できます）
2. 画面右下の「**📊 開封率チェッカー**」ボタンをクリック
3. 全ページを自動で読み取り（1ページずつ順に取得するため、ページ数が多いと数十秒かかります。
   「集計中… 3/10ページ」と進捗が表示されます）
4. ダッシュボードが表示される
   - 上部の期間フィルタで送信日を絞り込むと、すべてのグラフ・表が即座に再計算されます
   - 一度読み取ったデータは記憶されるので、期間を変えても再読み取りは不要です
   - 最新の状態に更新したいときは右上の「🔄 再取得」を押してください
5. 必要なら「明細をコピー」または「CSVダウンロード」でExcelなどに取り込む

## 集計の仕組みと注意点

- 一覧の各行の「既読／未読」表示を数えています。**開封率 = 既読数 ÷ 送信数**です
- メール一覧の**送信日時には年の表示がない**ため（例: `07/09 14:23`）、
  年は開封日時や現在日時から自動で推定しています（年をまたぐ場合も正しく処理）
- グラフの「開封率」の棒のうち、**薄い色の棒は送信数3件未満**で統計的な参考値です
- 「送信から開封までの経過時間」は、既読かつ送信日時・開封日時の両方が読み取れた行のみが対象です

## うまく動かないとき

| 症状 | 対処 |
|---|---|
| ボタンが表示されない | スカウトメール一覧ページ（URLに `/scoutMail/list` を含む画面）か確認し、ページを再読み込み（F5）する |
| 件数がおかしい | 「送信」タブを選んでから実行したか確認する |
| 「メールの行が見つかりませんでした」 | 一覧にメールが表示されている画面で実行する。表示されているのに出る場合は画面のHTML構造が変わった可能性があるため開発者に連絡 |
| 途中でエラーになる | 一度ページを再読み込みして再実行。続く場合はログイン切れの可能性があるので再ログイン |

## ファイル構成（開発者向け）

| ファイル | 役割 |
|---|---|
| `manifest.json` | 拡張機能の設定（対象サイトのみ。特別な権限は不要） |
| `content.js` | 本体。ページ巡回・行抽出・集計・ダッシュボード描画のすべて |
| `styles.css` | 起動ボタン・ダッシュボードの見た目 |

### 注意事項

- グルメキャリー側の画面デザイン変更（HTML構造の変更）があると、
  行の読み取りが動かなくなる場合があります。その場合は `content.js` の
  `parseRows()` の調整が必要です。
- 本拡張は社内利用を想定しています。
