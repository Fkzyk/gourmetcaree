# CLAUDE.md — gourmetcaree 開発メモ

このファイルは、過去の失敗を繰り返さないための記録。
**作業中に指摘・修正・失敗があったら、必ず「失敗と教訓ログ」に追記してからコミットすること。**

## プロジェクト概要

- グルメキャリー管理画面用のスカウトメール自動生成 Chrome 拡張機能
- Gemini(ブラウザ画面連携)でスカウト文面を生成し、フォームに自動入力する
- 最後の送信ボタンは必ず人間が押す設計(誤送信防止)

## ユーザーについて

- 非エンジニア。手順説明は番号付きで最小限・専門用語なしで
- 成果物はGitHubの説明をするより、**チャットに直接ファイル添付**して渡すのが確実
- 関連ツール: 業務週報メーカー(fkzyk/weekly-report-maker)— 週報作成は必ずそちらで行う
- **作成ファイルの命名規則**: `作成日_ファイル名(氏名)` 形式（例: `20260713_入社のご案内(重浪)`）。日付はYYYYMMDD

## 鉄則(過去の失敗から)

1. **このリポジトリにスカウト拡張と無関係のツールを入れない** — 新ツールは専用リポジトリを作る(例: 週報メーカーは fkzyk/weekly-report-maker に分離済み)
2. **文章の要約・清書はルールベース(キーワード抽出)でやらない** — 提出レベルの精度が必要なものはAI(Gemini)に清書させる
3. **帳票類の見た目は、必ずExcel原本やお手本PDFを確認してから作る** — 勝手な装飾(全実線の罫線・不要な枠)を足さない
4. **Word原本の再現は装飾込みで・レイアウトは変えない** — 辞令などの帳票を作り直すと飾り枠・ルビ・楷書体が消える。原本から構成要素(枠画像・フォント名・文字サイズ・ルビ)を抽出して再現し、必ずPDF等で目視確認してから渡す。**書字方向や配置を勝手に変えない**(ファイル内部の設定値より、ユーザーが知っている実物の見た目が正)
5. **メール下書きには「送信前チェック」を冒頭に入れる** — 日付未記入・添付忘れ・BCC追加漏れなど、送信時に埋める項目を本文冒頭に明記して渡す(該当行は送信前に削除してもらう)

## メール文面テンプレート

### 入社のご案内メール — {社宅段落}
転居を伴う入社者(社宅対象者)にのみ、以下の段落を挿入する。
**「社宅のご利用をご希望の場合は、」という表現は使わない**(転居入社では社宅利用が前提のため。2026-07-13にデフォルト文面として確定)。

```
また、ご転居を伴うご入社となりますので、「社宅利用申込のご案内」と「社宅システム入力マニュアル」も添付しております。
ご案内に記載の手順に沿って、お早めにお申し込みをお願いいたします。
※ご入居希望日は、入社日の10日前以降でご指定いただけます。
```

社宅対象でない方に送る場合はこの段落ごと削除する。
