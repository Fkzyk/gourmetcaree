# スカウトメール自動生成 - グルメキャリー版

グルメキャリー管理画面（kanri.gourmetcaree.jp）の候補者詳細ページで、
ボタン一つで候補者情報を読み取り、Gemini がスカウトメール（件名・本文）を
自動生成してフォームに入力してくれる Chrome 拡張機能です。

リクナビHRTech版の拡張機能をベースに、グルメキャリー用に移植したものです。

---

## できること

- スカウト送信画面の左下に青い丸ボタンを表示（目立たないようラベルなし）
- クリックすると候補者情報（職務要約・スキル・職歴・希望条件など）を自動で読み取り
- Gemini のタブが自動で開き、古川さんのスカウトメールプロンプトで文面を生成
- 生成された **件名（70文字以内）** と **本文（4000文字以内）** をフォームへ自動入力
- 文字数チェック付き（件名 70 / 本文 4000 を超えると赤字で警告）
- その日に「送信」ボタンを押した件数をボタン横のバッジに表示（毎日0時にリセット）
- **最後の送信ボタンは必ず人間が押す**設計です（誤送信防止）

## インストール方法（初回のみ・約3分）

1. このリポジトリの緑色の「**Code**」ボタン →「**Download ZIP**」でダウンロード
2. ZIPファイルを右クリック →「すべて展開」（好きな場所でOK。**展開後のフォルダは消さないこと**）
3. Chrome のアドレスバーに `chrome://extensions` と入力して開く
4. 右上の「**デベロッパーモード**」をONにする
5. 「**パッケージ化されていない拡張機能を読み込む**」をクリック
6. 手順2で展開した **gourmetcaree フォルダ** を選択
7. 一覧に「スカウトメール自動生成 - グルメキャリー」が表示されたら完了

> 更新版が出たときは、新しいZIPを同じ場所に展開し直して、
> `chrome://extensions` で拡張機能の「🔄（更新）」を押してください。

## 使い方

1. **Gemini（gemini.google.com）にログインしておく**（Googleアカウント）
2. グルメキャリー管理画面でスカウト送信画面を開く
3. 定型文を使う場合は、**先に**「定型文を選択」→「決定」を済ませる
   ※後から定型文を反映すると、生成した件名・本文が上書きされてしまうため
4. 画面左下の青い丸ボタンをクリック
5. Gemini のタブが自動で開き、生成が始まる（30秒〜1分ほど）
6. 生成が終わると元のページに自動で戻り、件名・本文が入力される
   ※本文欄に元から入っている「現在掲載中の求人情報はこちら」の求人リンクは、
   生成文の後ろに自動で残ります
7. 内容を確認して「送信」ボタンを押す（**送信ボタンは自動では押しません**）

## うまく動かないとき

| 症状 | 対処 |
|---|---|
| ボタンが表示されない | ページを再読み込み（F5）する。それでも出ない場合は、スカウトフォームのある画面か確認 |
| 「入力エリアが見つかりません」エラー | Gemini にログインしているか確認。Gemini のタブを一度閉じてやり直す |
| 生成が120秒でタイムアウト | Gemini が混んでいる可能性。少し待ってもう一度ボタンを押す |
| 件名/本文が入力されない | Geminiのタブに生成結果が残っているので、そこからコピーして手動で貼り付け。その旨を開発者に連絡（画面のHTML構造が変わった可能性） |
| 文字数オーバーの赤字警告 | もう一度生成ボタンを押して再生成する |

## ファイル構成（開発者向け）

| ファイル | 役割 |
|---|---|
| `manifest.json` | 拡張機能の設定（対象サイト・権限） |
| `background.js` | グルメキャリー ⇄ Gemini のタブ間通信を仲介 |
| `content_gourmet.js` | グルメキャリー側の本体。候補者情報の抽出、生成ボタンUI、結果のクリーニング、フォーム自動入力。**スカウトプロンプトはこのファイルの先頭に記載** |
| `content_gourmet_main.js` | フォームへの値注入（MAIN world・SPA対応） |
| `content_gemini.js` | Gemini側の本体。プロンプト投入・回答の取得 |
| `content_gemini_main.js` | Geminiの入力欄（Quill）への文字列挿入（MAIN world） |
| `popup.html` | ツールバーアイコンを押したときの使い方説明 |
| `styles.css` | ボタン・ステータス表示の見た目 |
| `docs/01_分析と要件定義.md` | 本拡張の分析・要件定義ドキュメント |

### プロンプトを変更したいとき

`content_gourmet.js` の先頭にある `SYSTEM_PROMPT = \`...\`` の中身を書き換えて、
`chrome://extensions` で拡張機能を更新（🔄）してください。

### 注意事項

- グルメキャリー側の画面デザイン変更（HTML構造の変更）があると、
  候補者情報の抽出やフォーム自動入力が動かなくなる場合があります。
  その場合は `content_gourmet.js` のセレクタ調整が必要です。
- 本拡張は社内利用を想定しています。
